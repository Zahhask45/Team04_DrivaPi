## Decisions

**Speed Conversion Location:**
- Option A: Perform conversion on STM32 microcontroller
- Option B: Send raw pulses to Raspberry Pi 5 and convert there
- **Decision:** Facilitators recommended STM32 approach as best practice. Team will proceed with this approach while maintaining compatibility with CAN communication.

**Cross-Compilation Strategy:**
- Option A: Continue debugging extensible SDK issues
- Option B: Compile Qt directly in AGL and import libraries (bypass)
- **Decision:** Team prefers to solve root cause (Option A) but will explore Option B as temporary workaround if needed. The bypass approach is being avoided for now to maintain proper development workflow and avoid technical debt. Success with AGL build and flash provides solid foundation for next steps.
